|| Progetto PSD Blueprint ||
	1. Obbiettivo del Sistema:
		-> Gestione segnalazioni urbane per un Comune;
		-> Operazioni:
			Registrazione, ricerca, aggiornamento stato, eliminazione, reportistica;
	2. Architettura dei Dati:
		-> ID (Codice identificativo univoco);
		-> Cittadino (Nome del segnalante);
		-> Categoria (Tipo problema:
										-> Illuminazione, rifiuti, strade);
		-> Descrizione: Descrizione del problema;
		-> Data: Data inserimento del problema;
		-> Urgenza (Livello gravita' del problema);
		-> Stato: (Aperta, Risoluzione, Revisione, Chiusa);
	3. Strategia tipo ADT (Indicizzazione Multipla);
		-> Struttura Principale:
			|| Lista collegata o Albero Binario (BST) ordinato per "Codice ID" per ricerche rapide;
		-> Indice urgenza:
			|| Coda con priorita' (Heap) per estrarre le segnalazioni piu' urgenti;
		-> Gestione Memoria:
			|| Utilizzo dei puntatori ai record originali per evitare duplicazioni di dati in memoria;
